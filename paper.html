Extending ICLRPoints: An Interactive System for Analyzing Research Effort

← Back to chart

Phu Le · March 9, 2026

Abstract

ICLRPoints is a metric that is used to analyze research effort based on conference publications in top-tier conferences. The original system provided only basic tools to view the publication activity and had limited interactive tools. In this project, I rebuilt ICLRPoints using updated data and added more interactive visualizations that make it easier to analyze publication activity across conferences and research areas. The result provides a flexible way to analyze research effort and compare publication activity over time.

Introduction

Conference publications are the primary way research is shared among the computer science community. Tools such as CSRankings and ICLRPoints provide methods for analyzing research output and effort by counting publications from top-tier conferences.

The original ICLRPoints project assigns scores to authors based on publications in these conferences. While this approach is useful, the website offers limited tools for exploring trends or interactively analyzing publication data.

This project extends the ICLRPoints framework by preserving the original methodology while introducing interactive visualization tools.

Data Sources

The system uses two main data sources: the DBLP bibliographic dataset and the CSRankings repository. DBLP provides comprehensive data on publications, including authors, titles, conferences, and years. CSRankings provides the list of top conferences and how they are grouped into research areas. Both of these datasets are found in the public CSRankings GitHub repository.

Methodology

The DBLP dataset is processed to find conference publications that match the CSRankings list of conferences. For each publication, the system extracts the authors, conference name, and publication year. They are then grouped to compute statistics such as the number of publications per author, the number of publications in each conference over time, and trends by research area.

To support the added interactive tools, the processed data is stored in a format that allows fast queries. ICLRPoints are computed by aggregating publication counts and fractional faculty over selected areas/conferences, then normalizing by a chosen baseline.

Assumption 1. Faculty members across different research areas devote roughly similar amounts of research effort per year.

Assumption 2. A large portion of academic research effort is shown in publications at top-tier conferences.

ICLRPoints Computation

The ICLRPoints metric estimates the average effort required to produce a publication in a research area. The total effort is approximated by the number of faculty in the area, while the total output is measured by the number of publications in top conferences.

The average effort required to produce a publication in an area is computed as

\[ \text{Average effort} = \frac{\text{Total faculty}}{\text{Total number of publications}} \]

\[ \text{ICLRPoints} = \frac{\text{Average effort}} {\text{Average effort of selected baseline}} \]

A higher ICLRPoints value indicates that producing a publication in that area requires more research effort on average.

Examples

Example 1: Machine Learning (baseline).

Suppose Machine Learning has 100 faculty and 1000 publications across conferences such as ICLR, ICML, and NeurIPS.

\[ \text{Average effort} = \frac{100}{1000} = 0.1 \]

Since Machine Learning is used as the baseline area, its ICLRPoints value is 1.

Example 2: Programming Languages (PLDI).

Suppose Programming Languages has 40 faculty and 80 publications across conferences such as PLDI and POPL.

\[ \text{Average effort} = \frac{40}{80} = 0.5 \]

\[ \text{ICLRPoints} = \frac{0.5}{0.1} = 5 \]

This means that producing a paper in a Programming Languages conference such as PLDI requires about five times more research effort per publication than producing a paper in a Machine Learning conference such as ICLR.

System Implementation

The system consists of two main components: a backend data pipeline and a web-based user interface.

The backend pipeline processes DBLP records, filters publications to the selected conferences, and computes statistics about publications. It is designed to be re-runnable as new DBLP and CSRankings snapshots become available, keeping the dataset up to date.

The web interface shows these statistics through a set of interactive visualizations. Users can filter by conference, research areas, year ranges, and toggle between area-level, conference-level, and trend views.

Differences from the Original ICLRPoints

The main improvements over the original ICLRPoints system include:

  • Interactive filters: Users can select or deselect parent areas, areas, and conferences using checkbox controls, with options to select or deselect all.
  • Year-range controls: Users can choose a start year and end year to compute results for a custom time period.
  • Multiple views of the metric:
    • Areas view: ICLRPoints by research areas
    • Conference view: ICLRPoints by individual conference
    • Trend view: ICLRPoints over time for each research area
  • Baseline selection: Users can select a baseline area or conference to normalize the points.
  • Detailed hover information: Graph tooltips show faculty count, publication count, and ICLRPoints values.

Future Work

Future improvements & features:

  • Institution and author views: Add views that allow users to compare individual researchers and departments, similar to the CSRankings system.
  • Citation-based metric: Incorporate citation data to measure research impact in addition to effort. Though, more recent publications typically have fewer citations than older ones.
  • Automated updates: Build a system that automatically updates the dataset and stores versioned results so they can be reproduced over time.

Conclusion

This project extends the ICLRPoints system by rebuilding the data pipeline with updated data and adding interactive visualization tools to better analyze publication data across conferences and research areas. The result provides a flexible and easy to use tools for analyzing research effort across conferences and research areas.

The data comes from CSRankings. Code available on GitHub.
